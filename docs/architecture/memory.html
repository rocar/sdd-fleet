Memory: a linked vault, not a curated store

The trap is letting a model curate memory — that puts non-determinism in the foundation the next spec builds on. Instead: one Obsidian vault with two .sdd/ levels — workspace plans above, each repo's specs and ADRs below — all plain markdown, so a human and Claude Code open the identical files.

# workspace/  — ONE Obsidian vault; member repos are git submodules (pinned SHAs)
#   .sdd/               ← WORKSPACE level · epic plan · contract design · lessons
#   authoriser/ .sdd/   ← REPO level · spec.md · acceptance.md · ADR-217.md
# authoriser/.sdd/spec.md:  Implements [PAY-1843] · payments.authorise@3.3.0 · [acceptance](./acceptance.md)
# backlinks are computed from these forward links — never written by hand
what sdd-fleet takes The discipline that falls out: standard markdown links, no [[wikilinks]] (they die on GitHub and to the gate parsers), cross-tree refs by stable ID, and a semantic index may sit on top for discovery but never feeds a gate. Memory is authored, never model-curated. Where this idea sits among the other three pillars: Four ideas, one boundary.
