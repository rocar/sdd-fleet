01

The boundary — where code decides, where the model judges, where a human signs off

One line governs the whole system: every consequence lives in code; the model judges only the irreducible residue; a human owns the blast radius. Every decision falls into one of the three columns below — the division itself is what an auditor examines.

A gate written as instructions to the model is a consequence handed to the model. So every consequence lives in a hook or a workflow — plain code, identical on a re-run.

Harness owns the consequence

  • Phase gates & the bounded loop
  • Survival-vote counting & routing
  • Semver & pinned-consumer checks
  • Blast-radius computation
  • Workspace dispatch & reconciliation
  • Fail-closed — anything missing or unverifiable blocks rather than allows

Model judges the residue

  • Drafting the plan, spec, tests, code
  • Per-criterion review & the adversarial pass
  • One soundness verdict per contested refutation
  • “Is this diff semantically breaking?”
  • Optional semantic index (regenerable)

Human owns the blast radius

  • Epic ratification, before work spreads to the repos
  • Contract-design sign-off
  • Money-movement & PII gates
  • Escalation resolution
  • Promoting a lesson into the record

The trail this ledger leaves re-derives the audit sentence — defined on the brief.
