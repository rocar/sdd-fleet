01

The boundary — where code decides, where the model judges, where a human signs off

One line governs the whole system: every consequence lives in code; the model judges only the irreducible residue; a human owns the blast radius. Each layer is given exactly the authority it can be trusted with — and no more.

Harness owns every consequence

  • phase gates, the bounded review loop, survival-vote counting (a concern stands unless refuted with evidence, and the counting is code)
  • semver + blast-radius computation, cross-repo dispatch
  • reproducible and fail-closed — identical on a re-run, and if anything is missing or unverifiable the gate blocks rather than allows

Model judges only the residue

  • the irreducible residue — the narrow, genuinely subjective calls no rule can settle
  • drafting a spec, per-criterion review, the single contested "is this diff breaking?" call
  • it never decides a consequence, and it never curates memory

Human owns the blast radius

  • the blast radius — how many dependent services a change reaches, and whether it touches money movement or PII
  • epic & contract ratification before fan-out; money / PII sign-off; escalation resolution
  • a human signature wherever the risk is too high to automate
A gate written as instructions to the model is a consequence handed to the model. So every consequence lives in a hook or a workflow — plain code, identical on a re-run.
02

Why the scaffolding exists

An AI agent generates plausible code non-deterministically — re-run the same prompt, get a different answer — and left unbounded, it also decides its own consequences: whether a concern matters, whether to merge. When a wrong call can move money or leak PII, “it usually gets it right” is not a control, and nothing about the run is reproducible or auditable. The response is not a better model. It is a deterministic harness over probabilistic generation — a control layer of plain code the model works inside.

Every input that could vary run-to-run — what a reviewer flags, how code is written — is the model's. Every consequence drawn from those outputs — does a concern survive, does a gate open, does this escalate — is the harness: pure code, identical on a re-run. The model judges; the harness decides what happens next. The residue left to the model is the narrow set of subjective calls no rule can settle — a draft, a review judgement, one contested decision. That single boundary is what makes the system auditable, and it is drawn — by colour — on every diagram across this site.

03

The decision ledger

Every decision in the system falls into one of three columns. The division itself is what an auditor examines.

Harness owns the consequence

  • Phase gates & the bounded loop
  • Survival-vote counting & routing
  • Semver & pinned-consumer checks
  • Blast-radius computation
  • Workspace dispatch & reconciliation

Model judges the residue

  • Drafting the plan, spec, tests, code
  • Per-criterion review & the adversarial pass
  • One soundness verdict per contested refutation
  • “Is this diff semantically breaking?”
  • Optional semantic index (regenerable)

Human owns the blast radius

  • Epic ratification, before work spreads to the repos
  • Contract-design sign-off
  • Money-movement & PII gates
  • Escalation resolution
  • Promoting a lesson into the record
▣ The audit line

From artifacts that were gated, not asserted — each repo's .sdd/ history, the pinned-SHA workspace commit, the registry's version history, Jira's sign-offs — the trail re-derives one sentence: “checkout's step-up change was built against catalog snapshot vX, authorise@3.3.0, additive per ADR-217, human sign-off by Y, contract tests green on a write-locked suite.”

Replay here means re-deriving that decision trail from the record, not regenerating identical code — defined in full on the brief.
04

What actually changes

The same change, under two control models. The difference is not the model or the tooling; it is who holds the decision at each step.
